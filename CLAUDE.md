# CodePulse

GitHub App + public URL analyzer that monitors codebase health on every push.
A BullMQ pipeline fans out to 5 parallel analysis workers, aggregates results into a
weighted HealthScore (0–100), and streams live updates to a Next.js dashboard.
Features: architectural drift detection via pgvector, agentic AI chat, debate mode,
root-cause investigator, cluster map, findings triage, MCP server, Stripe billing.

## ORM — Prisma

All database access goes through Prisma. No raw SQL except vector operations.

Schema is at `api/prisma/schema.prisma`. Run migrations from `api/`:
```bash
cd api
npx prisma migrate dev        # create + apply migration locally
npx prisma migrate deploy     # apply in production
npx prisma generate           # regenerate client after schema change
npx prisma studio             # DB browser UI at localhost:5555
```

**Neon requires two URLs:**
- `DATABASE_URL` — pooled via pgBouncer — all runtime queries
- `DIRECT_URL` — direct connection — `prisma migrate` only

Append `?connect_timeout=30&pool_timeout=30` to both.

## Current Prisma schema

```prisma
generator client {
  provider        = "prisma-client-js"
  previewFeatures = ["postgresqlExtensions"]
}

datasource db {
  provider   = "postgresql"
  url        = env("DATABASE_URL")
  directUrl  = env("DIRECT_URL")
  extensions = [vector]
}

model User {
  id          String   @id               // = Clerk user ID (sub claim)
  email       String   @unique
  plan        String   @default("free")
  repoLimit   Int      @default(3)
  clerkId     String?  @unique           // Phase 1
  githubLogin String?                    // Phase 1 — for webhook userId lookup
  createdAt   DateTime @default(now())
  repos       Repo[]
}

model Repo {
  id             String         @id @default(cuid())
  githubRepoId   Int                                  // NOT globally unique — scoped per user
  owner          String
  name           String
  installationId Int?                                 // nullable — URL-analyzed repos have no installationId
  source         String         @default("github_app") // "github_app" | "public_url" | "demo"
  defaultBranch  String         @default("main")
  isPublicDemo   Boolean        @default(false)
  lastAnalyzedAt DateTime?
  userId         String
  user           User           @relation(fields: [userId], references: [id])
  createdAt      DateTime       @default(now())
  snapshots      Snapshot[]
  fileAnalyses   FileAnalysis[]

  @@unique([userId, githubRepoId])       // composite unique — same repo can belong to different users
}

model Snapshot {
  id           String         @id @default(cuid())
  repoId       String
  repo         Repo           @relation(fields: [repoId], references: [id], onDelete: Cascade)
  commitSha    String
  healthScore  Float
  complexity   Float
  vulnCount    Int
  deadCode     Float
  coverage     Float
  driftScore   Float
  createdAt    DateTime       @default(now())
  fileAnalyses FileAnalysis[]

  @@index([repoId, createdAt])
}

model FileAnalysis {
  id         String                       @id @default(cuid())
  repoId     String
  repo       Repo                         @relation(fields: [repoId], references: [id], onDelete: Cascade)
  snapshotId String?
  snapshot   Snapshot?                    @relation(fields: [snapshotId], references: [id])
  filePath   String
  complexity Float                        @default(0)
  isDead     Boolean                      @default(false)
  driftScore Float?
  embedding  Unsupported("vector(1536)")?
  createdAt  DateTime                     @default(now())

  @@unique([repoId, filePath])
  @@index([repoId])
}

model DigestLog {
  id      String   @id @default(cuid())
  repoId  String
  sentAt  DateTime @default(now())
  summary Json
}
```

**pgvector raw SQL patterns:**
```javascript
// Insert embedding
await prisma.$executeRaw`
  UPDATE "FileAnalysis"
  SET embedding = ${JSON.stringify(vector)}::vector
  WHERE id = ${id}
`;

// Cosine similarity search
const results = await prisma.$queryRaw`
  SELECT "filePath", 1 - (embedding <=> ${JSON.stringify(queryVector)}::vector) AS similarity
  FROM "FileAnalysis"
  WHERE "repoId" = ${repoId}
  ORDER BY similarity DESC
  LIMIT 5
`;
```

## Critical rules — always follow these

### Code style
- **Backend (api/ + worker/): ESM** — `import`/`export` throughout. Never use `require()` or `module.exports`. Both `api/package.json` and `worker/package.json` have `"type": "module"`.
- **Dashboard: TypeScript strict mode**
- One file per worker type in `worker/workers/`
- Wrap all worker logic in try/catch — throw to mark job failed, never crash the process

### Source-aware file fetching (Phase 1+)
Workers must never call `getFileContent(installationId, ...)` directly. Always use the helpers:
```javascript
import { fetchFile } from '../lib/fetchFile.js';
import { fetchRepoTree } from '../lib/fetchRepoTree.js';

// Instead of: getFileContent(installationId, owner, repoName, path, ref)
const content = await fetchFile(job.data, filePath);

// Instead of: getRepoTree(installationId, owner, repoName, ref)
const paths = await fetchRepoTree(job.data);
```
`fetchFile` branches on `job.data.source`: `'github_app'` → App token; `'public_url'` | `'demo'` → public Octokit.

### Two analysis entry points
- **GitHub App push:** `webhook.js` → `fanOut({ source: 'github_app', installationId, ... })`
- **Public URL:** `analyze.js` → `fanOutPublic({ source: 'public_url', ... })` — no `installationId`

When a URL-analyzed repo later gets the App installed, the install callback upgrades it:
`upsert where (userId, githubRepoId)` → set `source = 'github_app'`, fill `installationId`. Same `repoId`, no duplicate rows.

### Repo uniqueness
`Repo` has `@@unique([userId, githubRepoId])` — NOT a global `@unique` on `githubRepoId`. Two different users can analyze the same public repo independently. All upserts must use:
```javascript
prisma.repo.upsert({ where: { userId_githubRepoId: { userId, githubRepoId } }, ... })
```

### Clerk auth
- `api/middleware/requireClerk.js` applies to all `/api/*` routes
- **Exception:** `/webhook` and `/health` — no auth
- `req.userId` = Clerk `sub` claim = `User.id` in DB
- `User.id` is always the Clerk user ID for Clerk-authed users; `'system'` for legacy webhook-only users

### Prisma
- All DB queries use Prisma except vector operations (`$executeRaw` / `$queryRaw`)
- Always run `npx prisma generate` after any schema change before starting the app
- `DATABASE_URL` (pooled) for all runtime queries — never use `DIRECT_URL` in app code
- `Unsupported("vector(1536)")` on the embedding column is correct — do not change it
- Schema lives at `api/prisma/schema.prisma` — NOT a top-level `prisma/` dir

### pgvector
- Embedding model: `text-embedding-3-small` — 1536 dimensions — do not change
- `<=>` returns cosine **distance** (0 = identical); similarity = `1 - distance`
- Default drift threshold: `0.72` — file is drifting if `maxSimilarity < 0.72`
- All vector reads/writes via `$executeRaw` / `$queryRaw` — Prisma doesn't support the `vector` type natively

### BullMQ
- Worker is always a **separate process** from API — never import worker logic into `api/index.js`
- Always call `worker.close()` on SIGTERM
- Queue names: `complexity-queue` `deadcode-queue` `vuln-queue` `coverage-queue` `drift-queue` `aggregator-queue` `insights-queue` (Phase 2) `cluster-queue` (Phase 5)
- Concurrency: `5` per worker
- Retry: `attempts: 5, backoff: { type: 'exponential', delay: 2000 }`
- Fan-out via `fanOut` / `fanOutPublic` in `api/services/queue.js` — never loop `queue.add()` calls

### GitHub App / Octokit
- Never cache the Octokit instance — installation tokens expire every hour
- `api/middleware/verifyGithub.js` — never modify
- File content from GitHub API is base64: `Buffer.from(data.content, 'base64').toString('utf8')`
- Use `crypto.timingSafeEqual` for signature comparison — never `===`

### Render free tier
- Web Service spins down after 15 min — Background Worker does not
- `GET /health` must return `{ ok: true }`
- UptimeRobot pings `/health` every 10 min to prevent spindown

## HealthScore formula

```
HealthScore (0–100) = 100
  - (complexity_penalty  × 0.25)   // escomplex avg normalized 0–100
  - (vuln_penalty        × 0.25)   // critical=10, high=5, moderate=2
  - (deadcode_penalty    × 0.15)   // (dead exports / total exports) × 100
  - (coverage_penalty    × 0.20)   // (1 - coverage_ratio) × 100
  - (drift_penalty       × 0.15)   // (flagged files / total files) × 100
```

## Drift detection

1. **On each push / URL analyze** — drift worker embeds each changed file, runs pgvector cosine similarity query (top 5 most similar existing files), stores raw `driftScore` = `MAX(similarity)`.
2. **Drift flagged** when `driftScore < driftThreshold` (default 0.72, configurable via `RepoSettings` in Phase 3).
3. Threshold applied **at read time** in routes — changing it doesn't require re-running the worker.
